## Exercice : Trouvez et utilisez la collection la plus adaptée pour chacun des cas suivants.

### Cas 1 : Inscriptions à une compétition de jeux vidéo
Objectif : Gérer la liste des participants à une compétition.

- Règles :
    - Chaque participant a un pseudo.
    - Un pseudo ne peut pas être utilisé plusieurs fois.
    - On doit pouvoir afficher rapidement la liste des participants par ordre alphabétique.

- Tâches :
    - Ajoutez plusieurs participants, dont certains en doublon.
    - Affichez la liste triée des participants.
    - Vérifiez si un participant précis est inscrit.

### Cas 2 : File d’attente dans un service client
Objectif : Simuler le traitement d’une file d’attente.

- Règles :
    - Chaque client a un nom.
    - Les clients arrivent les uns après les autres.
    - Le système doit traiter les clients dans l’ordre d’arrivée (FIFO).

- Tâches :
    - Ajoutez plusieurs clients à la file.
    - À chaque pression de la touche Entrée, affichez le nom du client en cours de traitement.
    - Affichez ensuite le nombre de clients restants.
    - Une fois la file vide, affichez : C'est l'heure de la pause.

### Cas 3 : Gestion des notes d’élèves
Objectif : Associer une note à chaque élève et les manipuler.

- Règles :
    - Chaque élève est identifié par son nom.
    - Chaque élève a une note sur 20.

- Tâches :
    - Ajoutez plusieurs élèves avec leurs notes respectives.
    - Modifiez la note du troisième élève ajouté.
    - Affichez la note du dernier élève ajouté.
    - Affichez la liste complète des élèves et leurs notes.
